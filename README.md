# FPSCookingPrototype

## 🎯 Цель проекта

Мини-прототип мобильной FPS-игры с фокусом на **быстрый и понятный gameplay loop**.

Игрок взаимодействует с объектами в мире, запускает процессы на станциях (например приготовление еды) и должен **следить за несколькими задачами одновременно**, реагируя на изменения их состояния.

Основная цель разработки — за короткое время (1–2 дня) собрать **рабочий, понятный и расширяемый прототип**, уделяя внимание ощущению взаимодействия и читаемости происходящего.

---

## 🧩 Реализовано

### Основной геймплей
- Подбор предметов
- Перенос предмета в руках
- Использование предмета на объектах
- Система взаимодействия через raycast от камеры

### Станции
- Конфигурируются через **ScriptableObject**
- Настраиваемые:
  - входной предмет
  - время приготовления
  - warning стадия
  - fail стадия
  - результат

Состояния станции:
- приготовление
- готово
- warning (скоро ошибка)
- ошибка (например «сгорело»)

Поддерживается **одновременная работа нескольких станций**.

---

### UI и визуальный фидбек
- World-space индикаторы прогресса
- Цветовая индикация стадий:
  - белый — готовится
  - жёлтый — warning
  - красный — ошибка
  - зелёный — готово
- Панель активных задач
- Hover-подсказки взаимодействия

---

### Управление (Mobile)
- виртуальный джойстик для движения
- свайп для поворота камеры
- поддержка multitouch
- разделение зон экрана (движение / камера / UI)

---

### Фидбек
- звуковые события станций
- частицы для приготовления и ошибки
- визуальные сигналы стадий

---

## 🏗 Архитектура

Проект разделён на логические модули:

- **Core**
- **Gameplay**
- **Systems**
- **UI**

Используется **data-driven подход через ScriptableObject**.

Добавление новых:
- предметов
- станций
- рецептов

происходит через конфигурацию, **без изменения кода**.

Архитектура намеренно упрощена, чтобы сохранить **скорость разработки и читаемость систем**, без избыточного усложнения.

## 🤖 Использование AI

AI использовался как вспомогательный инструмент для ускорения разработки и проверки решений.

Основные задачи, где он применялся:

- обсуждение архитектуры систем
- генерация базовых каркасов классов
- помощь в поиске и устранении ошибок
- быстрые итерации при улучшении UX
- оптимизация мобильного ввода и multitouch логики

Примеры задач:

- проектирование системы предметов и взаимодействия
- упрощение логики станций без сложной state machine
- реализация data-driven конфигураций через ScriptableObject
- улучшение обработки мобильного ввода
- идеи для визуального и звукового фидбека

AI позволил сократить время на рутинные задачи и сосредоточиться на **геймплейной логике и ощущении взаимодействия**.
